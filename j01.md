# OnVue CompTIA System Test Usability Challenge

Recently, I sat for my CompTIA Security+ Exam via an online proctored software that enables people to take exams online via Pearson's **OnVue desktop app**. 

Before taking the exam, I was required to download the OnVue app, authenticate and test my computer and network to see if I can have a seamless exam experience. I liked some of the **affordances** that the application provided such as the loading bar which showed the progress of the system test, which helped me know the status, therefore fulfilling Jakob Nielsen’s first UX heuristic about *Visibility of System Status*. 

<img width="1855" height="992" alt="image" src="https://github.com/user-attachments/assets/6fb7d933-c4f1-49c5-951e-754d97653c18" />

In addition, the application uses *minimalistic design* making it very clear and non-distractive to and gives the user the **ability to recognize** next steps, and also go to a previous page addressing Nielsen’s UX heuristics of *Recognition Rather Than Recall*, and *User Control and Freedom*. 

The challenge that I faced was when I reached the system test page. I encountered a Usability problem when I clicked the *“Test”* button. The application scanned my computer and returned apps and services that I should close. An example is in the screenshot below, and it only instructs a user to close the services mentioned without directing the user where to go and close for each of the services, except the Task Manager in Windows. I ended the running services, and restarted my computer; effectively starting the test all over again from the start *(logging in my CompTIA account, and getting the authentication code before authenticating in the OnVue desktop app) *.

Once again, at the test page, there were additional services which I had to close, for instance *vmware service*, *WSL*, *Hypervisor Services* which are not listed in the Task Manager. Therefore, it took me a bit of time to figure out how to stop these services. I googled and found that I can run *Win + r*, type *optional-features.exe* so that I could disable these services. After doing that and restarting my computer again, I was finally able to pass the system test. I was feeling a little bit frustrated by the experience because it **did not provide efficiency** as I ended up consuming half an hour, the directions were not effective and therefore it was not satisfying.

<img width="1800" height="826" alt="image" src="https://github.com/user-attachments/assets/15272be0-0054-455c-92a2-dcc994a00244" />

Some solutions I would suggest would be to have more guidance when users face the system testing errors, for example: *vmware dhcp service - go to Task Manager, right-click on this service and click “end task”.* This would address Nielsen’s heuristic of *Help and Documentation*, and it would make the experience more user-friendly.
